# Ejercicio 3.1 — Explicación del Workflow n8n

## Qué hace el workflow

Cada día laborable a las 8:00h, el workflow identifica los leads con status nuevo que llevan más de 48 horas sin ser gestionados. Si hay alguno, envía una alerta con el detalle de cada lead a un canal de notificaciones vía webhook. Si no hay ninguno, el workflow termina sin enviar nada.

---

## Nodos y su función

### 1. Schedule Trigger — Cada día laborable 8:00h
Dispara el workflow automáticamente con la expresión cron 0 8 * * 1-5:
- 0 8 = a las 8:00h
- 1-5 = solo de lunes a viernes (días laborables)

### 2. Code — Simular datos BigQuery
Simula la respuesta que vendría de una query a BigQuery o DuckDB. Genera un array de leads con distintos status, created_at y requested_amount. En producción, este nodo se reemplazaría por un nodo BigQuery o HTTP Request a la API de datos.

### 3. Code — Filtrar leads +48h
Aplica la lógica de negocio: filtra el array de leads manteniendo solo los que cumplen ambas condiciones:
- status === 'nuevo'
- Tiempo transcurrido desde created_at > 48 horas

Devuelve un objeto con el total de leads y la lista filtrada. Si no hay ninguno, devuelve total: 0.

### 4. IF — Hay leads sin gestionar?
Lógica condicional: evalúa si total > 0.
- TRUE = rama superior = construir y enviar alerta
- FALSE = rama inferior = terminar sin enviar nada

### 5. Code — Construir mensaje alerta
Formatea el mensaje de alerta con fecha en español y detalle de cada lead: ID, fecha de creación, fuente de captación, importe solicitado y horas de espera.

### 6. HTTP Request — Enviar alerta webhook
Envía el mensaje construido mediante POST al webhook configurado en webhook.site. En producción se sustituye por el webhook real de Slack, Teams o el sistema de alertas interno.

### 7. Code — Sin alertas hoy
Rama alternativa cuando no hay leads pendientes. Registra en log que no hay alertas y termina el workflow sin enviar ninguna notificación.

---

## Lógica condicional

El nodo IF es el núcleo de la lógica condicional. El enunciado especifica que si no hay ningún lead, no se envía nada. Esto se implementa con dos ramas:

Filtro -> IF (total > 0) -> TRUE  -> Construir mensaje -> Enviar webhook
                        -> FALSE -> Sin alertas hoy (termina)

## Nota sobre la fuente de datos

Segun el enunciado, no es necesario conectar una base de datos real. El nodo Simular datos BigQuery usa datos hardcodeados representativos del dataset real de iAhorro con los mismos campos: lead_id, created_at, source, requested_amount, status. En produccion este nodo se reemplazaria por una query directa a BigQuery.
