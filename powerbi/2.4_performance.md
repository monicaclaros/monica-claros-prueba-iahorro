# Ejercicio 2.4 — Performance

Ante un informe que tarda 15 segundos en cargar con un modelo importado de 2 millones de filas, el primer paso es identificar si el problema está en el modelo, las medidas o los visuales.

## Diagnóstico

1. Analizar tiempos de ejecución usando Performance Analyzer en Power BI para detectar qué visual o medida consume más tiempo (render, DAX o consulta).
2. Revisar el modelo de datos, verificando cardinalidad, relaciones innecesarias o bidireccionales y columnas no utilizadas que aumenten el tamaño del modelo.
3. Auditar medidas DAX, especialmente aquellas con iteradores (SUMX, FILTER) o cálculos complejos en contexto de filtro amplio.

## Herramientas

La herramienta principal es Performance Analyzer, complementada con DAX Studio para analizar consultas y tiempos de ejecución más detallados.

## Mejora del rendimiento

- Reducir columnas innecesarias y aplicar tipos de datos eficientes
- Evitar relaciones bidireccionales
- Simplificar medidas DAX y usar variables
- Crear una tabla de fechas adecuada para optimizar funciones de inteligencia temporal
- Reducir la complejidad de los visuales

El objetivo es minimizar el volumen de datos procesado y optimizar el cálculo en cada interacción.
